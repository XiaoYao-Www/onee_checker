## 1. 並行處理驗證 (CONC)

### CONC-001: rayon 線程池生命週期
| 項目 | 說明 |
|---|---|
| **目的** | 驗證 `ThreadPoolBuilder` 正確注入 rayon 全域池，且任務完成後無 thread leak |
| **步驟** | 1. `hash TEST --threads 4` 執行 2. 觀察 CPU 使用率分布 |
| **預期** | 所有 4 核心均參與運算；程式退出後無殘留 thread |

### CONC-002: 多線程 hash 計算一致性
| 項目 | 說明 |
|---|---|
| **目的** | 驗證 `compute_hashes_parallel` 在並行環境下的計算結果與單線程一致 |
| **步驟** | `hash TEST -a sha256 --threads 1` vs `--threads 8` → diff 兩個輸出檔 |
| **預期** | 兩個 hash 檔完全一致（排序後） |

### CONC-003: ProgressBar 與並行線程交互安全
| 項目 | 說明 |
|---|---|
| **目的** | 驗證 `indicatif::ProgressBar` 在 rayon 多線程中遞增時無 race condition |
| **步驟** | `hash TEST` (無 `-q`) → 觀察進度條遞增流暢無跳號/停滯 |
| **預期** | 進度條停在 878/878，無 panic |

### CONC-004: `HasherEnum` clone 線程安全
| 項目 | 說明 |
|---|---|
| **目的** | 驗證 `HasherEnum` 在多線程中 clone 並獨立計算時，結果隔離且正確 |
| **步驟** | 單元測試，8 線程各取 100 個檔案並行計算 MD5，比對單線程結果 |
| **預期** | 每個檔案的 hash 一致 |

### CONC-005: 誤差值變更後的線程池重建
| 項目 | 說明 |
|---|---|
| **目的** | 驗證連續兩次 `hash`（不同 `--threads` 值）時，rayon 全域池可重新初始化 |
| **步驟** | `hash TEST --threads 1` → 完成 → `hash TEST --threads 4` |
| **預期** | 兩次均正常完成 |

## 2. 安全性驗證 (SEC)

### SEC-001: Path Traversal — `..` 元件
| 項目 | 說明 |
|---|---|
| **目的** | 驗證 `sanitize_rel_path` 拒絕含 `..` 的路徑 |
| **步驟** | 建立 hash 檔含 `aabb... *../../etc/passwd` → `verify` |
| **預期** | stderr 顯示 "拒絕路徑逃脫 (..)"，不讀取外部檔案 |

### SEC-002: Path Traversal — 絕對路徑
| 項目 | 說明 |
|---|---|
| **目的** | 驗證拒絕 `/etc/passwd`、`C:\Windows\...` 等絕對路徑 |
| **步驟** | hash 檔含絕對路徑 → `verify` |
| **預期** | 正確拒絕 |

### SEC-003: Path Traversal — null byte 注入
| 項目 | 說明 |
|---|---|
| **目的** | 驗證拒絕 `\0` 字元（C 字串截斷攻擊） |
| **步驟** | `HashEntry::new("abc", "file\0.txt")` |
| **預期** | `Err(HashFileParseError)` |

### SEC-004: Path Traversal — Windows 裝置名稱
| 項目 | 說明 |
|---|---|
| **目的** | 驗證拒絕 CON、NUL、PRN 等保留名稱 |
| **步驟** | `sanitize_rel_path("CON/foo", root)` |
| **預期** | `Err(InvalidPath)` |

### SEC-005: Hash Parser — 非法 hex 字元
| 項目 | 說明 |
|---|---|
| **目的** | 驗證非 hex 字元的 hash 行被跳過並警告 |
| **步驟** | hash 檔含 `zzzz... *file.txt` → `verify` |
| **預期** | 跳過該行，stderr 顯示警告 |

### SEC-006: Hash Parser — DoS 行數上限
| 項目 | 說明 |
|---|---|
| **目的** | 驗證 `MAX_HASH_LINES = 10M` 硬限制 |
| **步驟** | 評估 parser 在大量行數下不 OOM（單元測試驗證 `.take(MAX_HASH_LINES)` 存在） |
| **預期** | 超過 10M 行後截斷，不 panic |

### SEC-007: Symlink 逃脫防護
| 項目 | 說明 |
|---|---|
| **目的** | 驗證指向目錄外的 symlink 不被遍歷 |
| **步驟** | 建立 symlink → 外部目錄 → `build_file_node` |
| **預期** | 外部檔案不在節點中 |

### SEC-008: 0-byte 檔案計算
| 項目 | 說明 |
|---|---|
| **目的** | 驗證空檔案 hash 計算不崩潰 |
| **步驟** | `compute_file_hash` 對 0-byte 檔案 |
| **預期** | 正確回傳已知 hash（SHA256: `e3b0c...`） |
